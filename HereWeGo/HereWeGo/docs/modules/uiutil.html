Module UIUtil

Provides useful functions and interfaces for the UI system.

I don't know how many events there are until the day of, and as a result I don't know how many event graphics we need to construct until the app is opened. The solution is dynamically generate the HTML at the start based on the data given to us. Industry experience (googling) has shown me that the fasted way to do this is to create what is known as a templating engine (ex. http://mustache.github.io/mustache.5.html) which uses string manipulation to create HTML pages and then renders it all at once. Of course I wanted fast and compact so I wrote my own found here: UIUtil.templateEngine

The rest of the UI component system is designed around using this templating engine to increase the reusability and modularity of the code written in each component. Everything in the UI is written in a class that extends UIUtil.UIItem. This class contains abstract methods for a HTML builder function which returns a string to put in the HTML, Javascript builder function which then binds listeners over the newly created elements, and multiple optional functions called on state changes (data update, time, etc.). These functiions are managed either by the top level script or a parent UIItem class (ex. SlideTabUI). All CSS is located in the index.css file, and classes in each component determine the effects applied to the HTML. I attempted to avoid inline styles for readability, but if needed they are a good debugging tool.

Data (events, schedule, etc.) is given to the UIItem through a parameter in UIUtil.UIItem.onInit or UIUtil.UIItem.onUpdate. This data must be generated from an instance of DataManage, and more detail on how this data is fetched and what it's contents are can be found in DataManage. Each UIItem has an array of UIUtil.RecvParams objects in UIUtil.UIItem.recvParams which specifies what kind of data is needed for the UIItem. This property must be a flat array of objects, so if a UIItem class has children UIItems with individual recvParams (ex. SlideTabUI) it must create a flat list of all the parameters during construction. This can easily be done using the UIUtil.combineParams function.

Const templateEngine

  • templateEngine(template: string, values: Object): string
    • Defined in UILib/UIUtil.ts:71

    Template engine function. Follows {{item}} syntax, meaning if a string "this is {{thing}}" is templated against the values {"thing" : 7} the output will be "this is 7". Based off of regular expressions and stackoverflow.

    Parameters

    • template: string

      The string to apply templating to

    • values: Object

      An object in format of {key: value} where '{{key}}' => value

    Returns string

    The templated string
